## <center>项目主线</center>  

本项目的目标不是单纯训练一个分类器，而是构建“物理模型扫描 + 严格机器学习判别 + 闭环定向 refine”的完整流程，用于研究 Nb2Se2O 对应 8 参数最小模型中的拓扑相界、拓扑平台和参数主导机制。

整体主线是：
1. 用 MagneticTB / Mathematica 在参数空间取样；
2. 通过 band + chern 计算得到物理标签；
3. 合并成总主表 master_dataset_all.csv；
4. 训练严格三层模型 A/B/C；
5. 用 joint 结果反向指导下一轮局部 refine；
6. 最终从闭环结果中提取 core / trivial / boundary 的代表集合和最终相图。

## <center>理论依据</center>  

1. 参数空间模型

当前工作的对象不是传统材料描述符，而是具有明确物理意义的 8 参数最小模型：
- e1, e2
- t1, t2
- r1, r2, r3, r4

因此这是一个“物理参数空间中的相结构学习问题”。

2. 为什么不是只做 brute-force

如果完全靠 brute-force，在高维参数空间中寻找相界和平台非常耗时，而且边界点很稀疏、很难正好扫到。

因此引入机器学习，目的不是替代理论计算，而是：
- 从已有样本中学习 topo / trivial / boundary 的结构；
- 反向告诉我们下一轮更值得算哪些区域；
- 提高局部 refine 的效率。

## <center>三层机器学习流程</center>  

1. Level A

目标：
- 判断一个样本是否是 physical insulator

作用：
- 先过滤掉明显不合理的金属/近隙闭合样本
- 为 topo 判断提供物理合理性约束

结论：
- Level A 一直较稳定，不是瓶颈
- 它在 joint 中相当于“物理门控层”

2. Level B

目标：
- 判断一个样本是否是 topo（target = is_topo_by_chern）

作用：
- 这是整个流程最核心的一层
- 直接决定 topo / trivial / boundary 的区分能力

为什么难：
- topo / trivial 本来就是参数空间中的模糊过渡
- 大量样本靠近相界
- 不是一个天然完全可分的任务

关键结论：
- 初期 strict 版 Level B 区分能力不够强
- 通过定向增样和多轮 refine 数据回流后，Level B 的 roc_auc / ap 明显提升
- 说明 ML 闭环确实有效

3. Level C

目标：
- 判断 chern 结果是否可靠

作用：
- 去除数值不稳定和伪拓扑结果
- 给 final topo 候选加“可信度”约束

结论：
- Level C 在 strict v2 后已经较稳定
- 在 joint 中承担可靠性过滤角色

## <center>joint的功能</center>  
joint 的作用，不是替代模型，而是把 A/B/C 三层结果融合成最终候选集合。

当前 joint_weighted 的核心形式为：
0.25 * p_insulator + 0.40 * p_topo + 0.35 * p_reliable

这样得到：
- topo_primary：较高置信 topo 候选
- topo_core：最高置信核心区
- trivial：高置信平庸区
- boundary：边界 / 过渡区

相比只用一个 topo 分类器，这种方式更严谨，因为：
- A 保证物理合理性
- B 强调拓扑倾向
- C 限制数值可靠性

## <center>数据构建与闭环 refine 流程</center>  
已经写好并成功运行：

整个流程为：
1. Mathematica / MagneticTB 生成原始 dat/hr
2. clean 清洗
3. band 计算 gap / 能带
4. chern 计算陈数与可靠性
5. merge 合并 band / chern / manifest
6. append 并入 master_dataset_all.csv
7. 训练 A/B/C
8. 跑 joint
9. 提取下一轮局部 refine 中心
10. 再回到 Mathematica 做局部精扫

这条闭环已经多轮验证可用。

## <center>各阶段结论</center>  

1. Stage5e / C=1 platform confirm

这一阶段最重要的结论是：
- 已经在参数空间中确认存在稳定的 C=1 拓扑平台核
- 这是后续机器学习闭环的物理基础

2. levelB_aug_batch001_lite

定位：
- 不是直接出最终物理图
- 而是专门为 Level B 提供高价值 hard / boundary 样本

结论：
- 新增样本进入训练后，Level B 明显变强
- 说明“ML 反向指导补样”的路线成立

3. next_refine_top30 / core12+boundary12

定位：
- 从 joint 中压缩出更高价值中心
- 分成 core 精扫与 boundary 窄窗口 refine

结论：
- Level B 再次提升
- 同时表明模型开始整体更偏 topo
- 原来适合旧模型的 joint 阈值不再适合新模型，需要重新校准

4. final_boundary_top20_refine

定位：
- 最后一轮偏 boundary 收束的 refine
- 使用 16 boundary + 4 core anchor

结论：
- 继续补数据的边际收益开始下降
- core / trivial 已经相对可控
- boundary 仍然较宽，说明参数空间本身存在较宽过渡带

## <center>为什么最终选择 t2 和 r3 作为主图坐标</center> 

1. 从 refine 设计上看，t2-r3 一直是主扫平面。
在多轮 Mathematica refine 脚本中，无论是：
- CORE_PLATFORM_FINE
- BOUNDARY_T2_R3_NARROW
- BOUNDARY_T2_R3_ULTRANARROW
主轴都优先围绕 t2 和 r3 展开，其他参数更多作为轻微扰动或辅助方向。

2. 从最终相图上看，t2-r3 平面上的结构最清楚。
最终 phase map 中：
- core / primary / boundary 的聚集关系在 t2-r3 上最直观；
- 可以清楚看到中心区和过渡区；
- trivial 对照也能同时展示。

3. 从 joint heatmap 上看，t2-r3 具有统计上的热点结构。
无论是 joint_weighted 还是 boundary_score_v3，在 t2-r3 平面上都呈现出明显的中心区热点，而不是全局随机均匀分布。

4. 从物理解释角度，t2 与 r3 承担了主导相界变化的角色。
结合整个闭环流程的经验，t2 和 r3 在多轮 refine 中持续表现出：
- 对 core 是否稳定存在有明显影响；
- 对 boundary 是否扩展/收缩有明显影响。

因此，最终选 t2-r3 做主图，不是因为“画起来方便”，而是因为：
- 它们在机器学习闭环里被持续证明是高敏感方向；
- 它们在最终相图与热图中都给出了最清楚的结构表达；
- 它们最能体现“平台核 + 宽过渡带”这一最终结论。

## <center>当前最终理解</center> 

1. core 已经相对稳定：
参数空间中确实存在高置信 topo 核心区，可以作为论文中“平台核”的主要展示部分。

2. trivial 已经可用：
可以作为论文中的对照区，也可证明模型不是只会报 topo。

3. boundary 仍然宽：
这是当前最重要的物理现象之一。
即使经过多轮边界 refine，boundary 仍然保持较宽，这说明：
- 它不只是数值噪声；
- 更像是参数空间中的真实过渡带。
因此 boundary 不一定要被继续机械压窄，而应该作为一个“值得解释”的物理结果。

## <center>机器学习流程的总依据</center> 

当前项目真正被验证的不是某个单一 accuracy 数字，而是以下整套逻辑：
1. 机器学习能从物理样本中学到参数空间结构；
2. Level B 能识别更像 topo / trivial / boundary 的区域；
3. joint 能把 A/B/C 融合成更可解释的候选集合；
4. 这些候选可以反向指导下一轮物理 refine；
5. 新物理样本回流后又能继续推强模型；
6. 多轮循环后，模型与物理结果一起逐渐收束。

也就是说，这是一条已经成立的：
“物理计算 -> 机器学习 -> 再物理 refine”
闭环路线。
