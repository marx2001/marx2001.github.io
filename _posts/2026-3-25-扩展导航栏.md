# Jekyll 博客导航栏修改总结

## 一、目的

本次修改的目标是解决 Jekyll 博客导航栏的几个问题：

1. 导航项过多时出现换行。
2. 导航栏改宽后出现左右贴边、不居中的情况。
3. `Home` 与左侧品牌名 `Mr.X's Blog` 之间距离过大。
4. 希望在最终效果上，让导航菜单整体相对品牌名向右平移一段距离，使视觉更协调。

---

## 二、涉及的文件名与目录位置

本次对话中涉及并确认过的关键文件如下。

### 1. 导航栏结构文件

**文件名：**
`navbar.html`

**目录位置：**
```text
_includes/navbar.html
```

**作用：**
控制导航栏的 HTML 结构，包括：

- 品牌名 `Mr.X's Blog`
- 各个菜单项
- 折叠按钮
- 外层容器与内部布局层级

---

### 2. 页面头部文件

**文件名：**
`head.html`

**目录位置：**
```text
_includes/head.html
```

**作用：**
决定页面实际加载哪些 CSS 文件。

**本次确认结果：**
该文件中实际加载的是：

```html
<link rel="stylesheet" href="{{"/assets/main.css" | relative_url }}">
```

这说明页面最终使用的是：

```text
/assets/main.css
```

因此，真正生效的样式入口是 `assets/main.css`，而不是最初修改的 vendor 目录下的 `_navbar.scss`。

---

### 3. 页面布局文件

**文件名：**
`default.html`

**目录位置：**
```text
_layouts/default.html
```

**作用：**
负责将 `head.html`、`navbar.html`、页面主体等部分拼装成完整页面。

**说明：**
该文件本身不直接决定导航栏样式来源，但决定了页面结构是如何组合的。

---

### 4. 最初尝试修改但未直接生效的主题样式文件

**文件名：**
`_navbar.scss`

**目录位置：**
```text
assets/vendor/startbootstrap-clean-blog/scss/_navbar.scss
```

**作用：**
这是主题源码中与 `#mainNav` 相关的 SCSS 文件。

**结论：**
虽然这里确实存在 `#mainNav` 样式，但本次站点实际加载的是 `assets/main.css`，因此直接修改这个文件并没有在页面上体现出来。

---

### 5. 最终真正生效的样式文件

**文件名：**
`main.css`

**目录位置：**
```text
assets/main.css
```

**说明：**
页面实际加载的就是这个文件，所以最终的覆盖样式需要追加到这里，或者追加到它对应的源文件 `assets/main.scss` 中。

如果项目中存在：

```text
assets/main.scss
```

那么更推荐修改它；如果没有，或者为了快速验证效果，可以直接在 `assets/main.css` 末尾追加覆盖样式。

## 三、问题排查过程

### 1. 初始问题：导航栏换行

一开始观察到导航栏项目较多时发生换行，初步判断是导航栏可用宽度不足。

在 `navbar.html` 中原始结构为：

```html
<div class="container">
```

这是 Bootstrap 的固定宽度容器，当导航项较多时容易导致一行放不下，从而换行。

---

### 2. 第一次尝试：改成 `container-fluid`

将：

```html
<div class="container">
```

改为：

```html
<div class="container-fluid">
```

#### 结果

优点：

- 导航栏可用宽度增大；
- 缓解了换行问题。

缺点：

- 导航整体变成左右贴边；
- 菜单与品牌名分得太开；
- 视觉上像“两边对齐”，缺少居中感。

---

### 3. 第二次尝试：增加 `nav-inner`

为了解决“更宽但又要居中”的问题，采用了新的结构思路：

- 外层仍然使用 `container-fluid`
- 内层增加一个自定义容器 `nav-inner`

其作用是：

- 允许导航区域比默认 `container` 更宽；
- 同时通过 `max-width + margin: 0 auto` 保持整体居中；
- 避免 `container-fluid` 导致的左右贴边问题。

---

### 4. 第三次排查：为什么改 `_navbar.scss` 没有效果

虽然通过搜索 `#mainNav` 找到了：

```text
assets/vendor/startbootstrap-clean-blog/scss/_navbar.scss
```

但多次修改后页面没有任何变化。

继续排查后，在：

```text
_includes/head.html
```

中确认页面实际加载的是：

```html
<link rel="stylesheet" href="{{"/assets/main.css" | relative_url }}">
```

#### 由此得出的关键结论

真正生效的样式入口是：

```text
assets/main.css
```

因此：

- 直接改 vendor 目录里的 `_navbar.scss`
- 并不一定会反映到页面
- 必须改最终实际加载的主样式文件

这也是前面“样式写了但页面完全没变化”的根本原因。

---

## 四、最终采用的结构修改方案

### 修改文件

```text
_includes/navbar.html
```

### 主要修改内容

#### 1. 将外层容器改为 `container-fluid`

由：

```html
<div class="container">
```

改为：

```html
<div class="container-fluid">
```

---

#### 2. 新增 `nav-inner`

在 `container-fluid` 内再包一层：

```html
<div class="nav-inner">
```

用于控制：

- 最大宽度
- 水平居中
- 内容排列

---

#### 3. 去掉 `ml-auto`

原本菜单列表使用：

```html
<ul class="navbar-nav ml-auto">
```

这会让菜单整体被推到最右边，导致 `Home` 与品牌名距离过大。

最终改为：

```html
<ul class="navbar-nav">
```

### 最终思路

1. 设置 `nav-inner`：
   - 保证更宽但不贴边
   - 整体居中

2. 设置 `navbar-brand`：
   - 不换行

3. 设置 `navbar-collapse`：
   - 不再自动占满剩余宽度

## 七、最终采用的样式代码

以下为最终实际生效的样式方案。

注意：在最后微调时，`margin-left` 最终被设为：

```css
6rem
```

这是最终满意的参数。

```css
@media (min-width: 992px) {
  #mainNav .nav-inner {
    width: 100%;
    max-width: 1400px;
    margin: 0 auto;
    display: flex;
    align-items: center;
  }

  #mainNav .navbar-brand {
    white-space: nowrap;
    flex-shrink: 0;
  }

  #mainNav .navbar-collapse {
    display: flex !important;
    flex-grow: 0 !important;
    justify-content: flex-start !important;
    margin-left: 6rem !important;
  }

  #mainNav .navbar-nav {
    margin-left: 0 !important;
  }

  #mainNav .navbar-nav .nav-link {
    white-space: nowrap !important;
    padding-left: 12px !important;
    padding-right: 12px !important;
  }
}
```

四、最终结构方案（navbar.html）
-------------------------------
最终导航栏结构修改的核心思路是：

1. 外层使用：
   <div class="container-fluid">
   以获得比默认 container 更大的横向空间。

2. 在其中加入：
   <div class="nav-inner">
   作为真正的内容容器，用于：
   - 限制最大宽度；
   - 实现整体水平居中；
   - 避免导航贴边。

3. 菜单列表使用：
   <ul class="navbar-nav">
   而不是：
   <ul class="navbar-nav ml-auto">
   因为 ml-auto 会把菜单整体推向右边，导致 Home 离品牌名过远。

4. 最终 navbar.html 的关键结构是：
   - nav#mainNav
   - .container-fluid
   - .nav-inner
   - .navbar-brand
   - .collapse.navbar-collapse
   - .navbar-nav
